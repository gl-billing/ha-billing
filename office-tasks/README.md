# HA Office Task & Calendar Management

Separate app from **HA Billing** (`../web/`). Same firm Google sign-in pattern; **different spreadsheet** and Vercel project.

## Architecture

```
Staff browser  →  tasks-web (Next.js)  →  Google Sheets API (Master Tasks, Hearings & Events, Employees)
                                      →  Apps Script Web App (overviews, PDF, calendar sync, email reminders)
```

| Layer | Responsibility |
|-------|----------------|
| **Spreadsheet** | Source of truth: tasks, events, employees |
| **Apps Script** (`apps-script/Code.gs`) | Sidebar logic you already have: overviews, triggers, calendar, reminders, tax deadlines |
| **tasks-web** | Modern UI: Today, tasks, events, search — read/write source tabs |

Keep **overview sheets** (Today / Weekly / Monthly) generated by Apps Script for now. The web app can call **Refresh All Overviews** via the bridge API until those views are rebuilt in React (Phase 2).

## Spreadsheets

Create a **new** Google Spreadsheet (do not mix with billing):

- Suggested title: `HA Office Task & Calendar Management Master System`
- Paste your Apps Script into the project → **Task System → Setup / Repair Workbook**

Env var for the web app:

```bash
GOOGLE_SPREADSHEET_ID=<tasks-spreadsheet-id>
```

## Phase 1 (tasks-web — started in `../tasks-web/`)

- [x] Today dashboard (counts from Master Tasks + Hearings & Events)

## Deploy (separate Vercel project)

1. New repo or monorepo folder `tasks-web` as root on Vercel
2. Env: same `GOOGLE_CLIENT_*`, new `GOOGLE_SPREADSHEET_ID`, `NEXTAUTH_*`, `ALLOWED_EMAILS`, `ADMIN_EMAILS`
3. Apps Script: deploy Web App → `TASKS_APPS_SCRIPT_WEB_APP_URL` + secret
4. Google OAuth: add tasks app URL to authorized origins/redirects

## Apps Script setup

1. New Apps Script project bound to the **tasks** spreadsheet (or paste `Code.gs` + `WebAppApi.gs`)
2. Run **Task System → Setup / Repair Workbook** once
3. Deploy Web App (for bridge): Execute as Me, Anyone with link + secret
4. **Install Automation Triggers** from menu (or web button later)
